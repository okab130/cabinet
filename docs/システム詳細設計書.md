## 5. データモデル

### 5.1 Invoice（帳票）

```typescript
type Invoice = {
  id: string;                    // ドキュメントID
  type: 'send' | 'receive';      // 送信 / 受信
  docType: 'invoice'             // 帳票種類
         | 'delivery_note'
         | 'quotation'
         | 'purchase_order'
         | 'receipt';
  invoiceNumber: string;         // 帳票番号（例: INV-2024-001）
  partnerId: string;             // 取引先ID
  partnerName: string;           // 取引先名（非正規化）
  amount: number;                // 税抜金額
  taxAmount: number;             // 消費税額
  totalAmount: number;           // 合計金額（税込）
  issueDate: string;             // 発行日（YYYY-MM-DD）
  dueDate: string;               // 支払期限（YYYY-MM-DD）
  status: InvoiceStatus;         // ステータス
  description: string;           // 件名・摘要
  stampedAt?: string;            // 電子押印日時（ISO8601）
  stampedBy?: string;            // 押印者ID
  createdAt: string;
  updatedAt: string;
  createdBy: string;
};

type InvoiceStatus =
  | 'draft'             // 下書き
  | 'pending_approval'  // 承認待ち
  | 'sent'              // 送信済み
  | 'received'          // 受信済み
  | 'stamped'           // 電子押印済み
  | 'approved'          // 承認完了
  | 'rejected'          // 差戻し
  | 'cancelled';        // キャンセル
```

### 5.2 ApprovalRequest（承認申請）

```typescript
type ApprovalRequest = {
  id: string;
  type: 'invoice_send' | 'invoice_receive';
  invoiceId: string;
  title: string;
  description: string;
  requesterId: string;
  requesterName: string;
  currentStepIndex: number;      // 現在のステップ（0始まり）
  steps: ApprovalStep[];
  status: 'pending' | 'approved' | 'rejected' | 'cancelled';
  createdAt: string;
  updatedAt: string;
};

type ApprovalStep = {
  stepIndex: number;
  approverId: string;
  approverName: string;
  status: 'pending' | 'approved' | 'rejected';
  comment?: string;
  actedAt?: string;
};
```

## 6. 状態管理設計（Zustand）

### 6.1 authStore

```typescript
// 管理する状態
interface AuthState {
  currentUser: User | null;
  isAuthenticated: boolean;
  login: (email: string, password: string) => void;
  logout: () => void;
}
```

**ログイン処理**: メールアドレスでモックユーザーを検索。一致しない場合は `admin@example.com` にフォールバック。

### 6.2 invoiceStore

```typescript
interface InvoiceState {
  invoices: Invoice[];
  approvals: ApprovalRequest[];
  partners: Partner[];

  // 帳票操作
  addInvoice: (invoice: Invoice) => void;
  updateInvoice: (invoice: Invoice) => void;
  stampInvoice: (id: string, userId: string) => void;

  // 承認操作
  approveStep: (approvalId: string, comment: string) => void;
  rejectStep: (approvalId: string, comment: string) => void;

  // 取引先操作
  addPartner: (partner: Partner) => void;
  updatePartner: (partner: Partner) => void;
  deletePartner: (id: string) => void;
}
```

**stampInvoice**: ステータスを `stamped` に変更し `stampedAt`・`stampedBy` を記録。  
**approveStep**: 現在ステップを `approved` に更新。全ステップ完了時は申請全体を `approved` に、次ステップがある場合は `currentStepIndex` をインクリメント。

**バックエンドAPI（オプション）**
- Java 21
- Maven 3.9.x
- Git

### 10.2 セットアップ

```bash
git clone https://github.com/okab130/cabinet.git
cd cabinet
npm install --legacy-peer-deps
```

> `--legacy-peer-deps` が必要な理由: `@react-pdf/renderer` が React 18 との peer dependency 競合を持つため。

### 10.3 フロントエンド開発サーバー起動

```bash
npm run dev
# → http://localhost:5173/
```

### 10.3.1 バックエンドAPI起動（Java 21 + Maven 必須）

```bash
cd backend
mvn spring-boot:run
# → http://localhost:8080
```

**主要エンドポイント確認**:

| URL | 内容 |
|-----|------|
| `GET http://localhost:8080/api/health` | ヘルスチェック |
| `GET http://localhost:8080/api/invoices` | 帳票一覧 |
| `GET http://localhost:8080/api/partners` | 取引先一覧 |
| `GET http://localhost:8080/api/approvals` | 承認申請一覧 |

### 10.4 本番ビルド

```bash
npm run build
# → dist/ に出力（HTML + CSS + JS バンドル）
```

## 11. 既知の制約・今後の課題

| 項目 | 状況 | 対応方針 |
|------|------|---------|
| PDF日本語フォント | Helveticaのため日本語が文字化けする場合あり | `Font.register()` で Noto Sans JP を登録 |
| バックエンドAPI ✅ | インメモリ実装済み。本番はFirestore接続 | Zustand store を /api/* 呼び出しに切り替え |
| ファイルアップロード | UIのみ（実ファイル保存なし） | Firebase Storage に接続 |
| Peppol送受信 | 未実装 | Peppol Access Point 経由の XML 送受信 |
| メール通知 | 未実装 | Firebase Functions + SendGrid |
| 監査ログ | 未実装 | Firestore `auditLogs` コレクションへの操作記録 |
| ページリロード | Zustandはメモリ管理のためリロードでデータリセット | バックエンドAPI接続 or localStorage永続化 |
