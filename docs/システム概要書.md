# システム概要書

**システム名**: InvoiceFlow — 電子取引管理システム  
**バージョン**: 1.0.0  
**作成日**: 2026年3月26日  
**対象読者**: 経営層・プロジェクトオーナー・業務担当者

---

## 1. システムの目的

InvoiceFlow は、企業間の電子帳票（請求書・納品書・見積書・発注書）の送受信・管理・承認を一元化するWebアプリケーションです。  
WingArc1st 社の「invoiceAgent 電子取引」をリファレンスに、**電帳法（電子帳簿保存法）対応**・**インボイス制度対応**・**社内承認ワークフロー**を中核機能として設計されています。

---

## 2. 解決する業務課題

| 課題 | 現状 | InvoiceFlow 導入後 |
|------|------|------------------|
| 紙帳票の郵送コスト | 1通あたり約80〜150円のコスト | 電子送受信でコストほぼゼロ |
| 受領後の承認フローが属人的 | メール・口頭での承認依頼 | システム上で可視化・自動化 |
| 電帳法対応の保存管理 | Excelや紙での台帳管理 | システムが自動で証跡・タイムスタンプを管理 |
| インボイス番号の確認作業 | 手動で国税庁サイトを確認 | 取引先マスタに登録・一元管理 |
| 帳票の所在把握 | ファイルサーバーが散在 | フォルダツリーで統合管理 |

## 4. 主要機能一覧

### 4.1 コア機能

| 機能 | 説明 | 対応法令 |
|------|------|---------|
| **帳票送信** | 請求書・納品書・見積書・発注書を電子送信 | 電帳法・インボイス制度 |
| **帳票受信** | 取引先からの帳票を受信・管理 | 電帳法 |
| **電子押印** | 受領確認の電子スタンプ（タイムスタンプ付き） | 電帳法（真実性確保） |
| **承認ワークフロー** | 多段階の社内承認フロー（可視化付き） | 内部統制 |
| **Peppol対応** | 国際電子請求書標準フォーマット送受信 | デジタルインボイス推進 |

### 4.2 管理機能

| 機能 | 説明 |
|------|------|
| **取引先管理** | インボイス登録番号・連絡先・有効/無効管理 |
| **文書管理** | フォルダツリー形式でのファイル整理・検索 |
| **帳票出力** | PDF・Excel形式での帳票生成・ダウンロード |
| **ユーザー管理** | ロールベースアクセス制御（RBAC） |

### 4.3 分析機能

| 機能 | 説明 |
|------|------|
| **ダッシュボード** | 月別送受信件数・ステータス内訳・KPI表示 |
| **処理待ち一覧** | 承認待ち・確認待ちの帳票を優先表示 |

## 5. ユーザーロール

| ロール | 説明 | 主な操作権限 |
|--------|------|------------|
| **管理者 (admin)** | システム全体を管理 | 全機能 + ユーザー管理 |
| **マネージャー (manager)** | 承認権限を持つ責任者 | 全機能 + 承認操作 |
| **一般ユーザー (user)** | 日常業務担当者 | 帳票送受信・申請 |
| **閲覧者 (viewer)** | 参照のみ | 閲覧のみ |

## 6. 法令対応

### 電子帳簿保存法（電帳法）
- 受領した電子取引データの保存要件に対応
- タイムスタンプ（電子押印）による真実性確保
- 検索機能による可視性確保

### インボイス制度（適格請求書等保存方式）
- 取引先の適格請求書発行事業者登録番号（T + 13桁）管理
- 帳票への登録番号記載
- 仕入税額控除のための保存管理
